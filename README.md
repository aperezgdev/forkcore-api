# ForkCore API

## Alcance del proyecto

Este repositorio se creó como proyecto sandbox para probar y validar el
[harness template de workflow](https://github.com/aperezgdev/harness-template-workflow).

Su único propósito fue servir como caso de uso real del flujo definido por el
template (diseño funcional -> arquitectura -> acceptance tests -> TDD -> BDD ->
review), por lo que queda marcado como completado.

## Conclusión

El resultado ha sido satisfactorio: el flujo se ha podido ejecutar de punta a
punta sobre un proyecto real. Como contrapartida, el template genera demasiada
documentación para proyectos pequeños y consume un volumen de tokens
desproporcionado, por lo que conviene reservarlo para proyectos de tamaño medio
o grande donde ese sobrecoste se justifique.
